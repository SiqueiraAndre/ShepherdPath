## 🚀 The Problem: The "Sunday Night" Bottleneck

Before ShepherdPath, attendance registration relied on generic tools (such as standard online forms) that generated an exhausting workflow:

- **Manual Cleanup:** Needing to delete dozens of empty columns generated by the branching logic of the forms.
- **Complex Sorting:** Wasting time organizing hundreds of students by catechist and alphabetical order every week.
- **Manual Export:** Generating manual reports or PDFs to send to guardians via WhatsApp.

## ✨ The Solution: Purposeful Automation

The system acts on the root of the problem, structuring the data right from the check-in moment:

- 📱 **Smart QR Code Check-in:** A *mobile-first* interface where students register their attendance at the church entrance.
- 🎯 **Dynamic Stage Filtering:** Uses **Alpine.js** to show only the catechists for the selected stage (1st or 2nd), preventing input errors.
- 📄 **Automated Reports:** A background **Job** processes the week's attendance and generates clean, organized PDFs per class, ready for distribution.
- ⚡ **Robust Performance:** Sessions and queues managed via **Redis**, ensuring speed even with hundreds of simultaneous accesses during mass.

## 🛠️ Tech Stack

- **Backend:** Laravel 12 (PHP 8.2+)
- **Database:** PostgreSQL (Robust and performant relationships)
- **Cache/Queue:** Redis
- **Admin Panel:** FilamentPHP v4
- **Frontend:** Tailwind CSS & Alpine.js
- **Environment:** Docker (Laravel Sail)

## 📦 Installation

The project runs on **Laravel Sail**, which ensures a clean development environment with support for the latest versions using Docker *containers*.

1. **Clone the repository:**
   ```bash
   git clone https://github.com/SiqueiraAndre/shepherd-path.git
   cd shepherd-path
   ```

2. **Create the environment file:**
   ```bash
   cp .env.example .env
   ```

3. **Initialize the Sail instances:**
   If the local machine does not have Composer/PHP installed, you can use the Docker image:
   ```bash
   docker run --rm \
       -u "$(id -u):$(id -g)" \
       -v "$(pwd):/var/www/html" \
       -w /var/www/html \
       laravelsail/php84-composer:latest \
       composer install --ignore-platform-reqs
   ```

4. **Start the Docker environment:**
   ```bash
   ./vendor/bin/sail up -d
   ```

5. **Initial Database Configuration (Key and Migrations with real seeders):**
   ```bash
   ./vendor/bin/sail artisan key:generate
   ./vendor/bin/sail artisan migrate --seed
   ```

6. **Start the queues and workers:**
   ```bash
   ./vendor/bin/sail artisan queue:work
   ```

7. **Access the application in your browser:**
   - **System (Check-in):** [http://localhost](http://localhost)
   - **Admin Panel:** [http://localhost/admin](http://localhost/admin)

## ⚙️ How It Works

1. **Configuration:** The coordinator accesses the admin panel to register the *Stages*, *Catechists*, and *Masses*.
2. **Attendance Registration:** At the church door, students scan the QR Code to mark their attendance, using a simple, foolproof interface.
3. **Secure Processing:** The system validates the received data and stores it securely in PostgreSQL.
4. **Scheduled Delivery:** Every Monday (or at a configured time), the system consolidates the data, generates PDFs for each class, and delivers them via email or WhatsApp, without a single Excel file needing to be touched.
